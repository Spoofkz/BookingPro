# PRD-00: End-to-End Scenarios, Coverage & Verification

This folder implements the governance system defined by **PRD-00**:

- every scenario has exactly one Owner milestone
- dependencies are explicit
- traceability exists from scenario -> AC -> tests -> evidence
- milestone readiness can be computed from evidence (not estimates)

## Governance rules enforced by script

- No duplicate scenario IDs
- Every scenario has one valid owner milestone
- Dependencies reference valid milestones
- Scenario status must be `0`, `50`, or `100`
- Every scenario appears in both traceability matrices
- Traceability Matrix B verdict must match rule logic
